# UX Spec: realtime quiz MVP

Документ предназначен для переноса в Figma/Miro. Это инвентарь экранов,
состояний и основных потоков без финальной визуальной стилистики.

## Экраны

### 1. Лендинг

Назначение: объяснить ценность продукта и направить пользователя к входу,
регистрации или подключению по коду.

Ключевые элементы:

- Название продукта и короткое позиционирование.
- Поле ввода кода сессии.
- CTA `Join`.
- CTA `Login` / `Sign up`.
- Блок с кратким описанием сценария для организатора.

Состояния:

- Loading: проверка текущей сессии пользователя.
- Empty: первый визит без авторизации и без введенного кода.
- Error: неверный или истекший код сессии.
- Active: код введен, доступна отправка формы.

### 2. Регистрация / логин

Назначение: создать пользователя или войти в существующий аккаунт.

Ключевые элементы:

- Email.
- Пароль.
- Display name.
- Выбор роли при регистрации: `ORGANIZER` или `PARTICIPANT`.
- Переключение login/sign up.
- Сообщение об ошибке авторизации.

Состояния:

- Loading: отправка формы.
- Empty: форма без заполненных данных.
- Error: неверные учетные данные, занятый email, слабый пароль.
- Active: валидная форма, доступна отправка.

### 3. Кабинет организатора

Назначение: управлять квизами, запускать сессии и смотреть историю.

Ключевые элементы:

- Список квизов.
- Поиск/фильтр по категории.
- Кнопка создания квиза.
- Действия квиза: edit, duplicate, start session.
- Ссылка на историю проведенных сессий.

Состояния:

- Loading: загрузка квизов.
- Empty: квизов еще нет, показан CTA создания.
- Error: ошибка загрузки списка.
- Active: список квизов с доступными действиями.

### 4. Конструктор квиза

Назначение: задать общие параметры квиза и управлять вопросами.

Ключевые элементы:

- Title.
- Description.
- Category.
- Default time limit.
- Список вопросов с порядком.
- Добавление вопроса.
- Save / Publish draft action.

Состояния:

- Loading: загрузка существующего квиза.
- Empty: новый квиз без вопросов.
- Error: ошибка сохранения или валидации.
- Active: форма редактирования с autosave/status-индикатором.

### 5. Конструктор вопроса

Назначение: создать или отредактировать вопрос и варианты ответа.

Ключевые элементы:

- Тип вопроса: `SINGLE` / `MULTIPLE`.
- Текст вопроса.
- Image URL.
- Time limit override.
- Points.
- Список answer options.
- Флаг `isCorrect` для каждого варианта.
- Reorder options.

Состояния:

- Loading: загрузка вопроса.
- Empty: новый вопрос без вариантов.
- Error: нет корректного ответа, пустой текст, ошибка сохранения.
- Active: вопрос валиден, доступно сохранение.

### 6. Лобби / комната по коду

Назначение: собрать участников перед стартом live-сессии.

Ключевые элементы:

- 6-символьный код сессии.
- QR-код или join-ссылка.
- Список подключенных участников.
- Для организатора: Start quiz.
- Для участника: nickname и статус ожидания.

Состояния:

- Loading: подключение к сессии.
- Empty: участников пока нет.
- Error: сессия не найдена, уже завершена или соединение потеряно.
- Active: участники подключаются в реальном времени.

### 7. Экран вопроса участника

Назначение: дать участнику ответить на текущий вопрос.

Ключевые элементы:

- Текст вопроса.
- Изображение вопроса, если есть.
- Таймер.
- Варианты ответов.
- Индикатор выбранных вариантов.
- Submit или auto-submit по таймеру.

Состояния:

- Loading: ожидание следующего вопроса.
- Empty: вопрос еще не запущен.
- Error: ответ не принят или соединение потеряно.
- Active: вопрос открыт, таймер идет.

### 8. Экран вопроса на сцене / проекторе

Назначение: показать вопрос аудитории без приватных действий участника.

Ключевые элементы:

- Номер вопроса.
- Текст вопроса.
- Изображение вопроса.
- Таймер.
- Варианты ответов без интерактивного выбора.
- Счетчик полученных ответов.

Состояния:

- Loading: подготовка вопроса.
- Empty: ожидание старта.
- Error: сессия потеряна или вопрос недоступен.
- Active: вопрос показан, таймер и счетчик обновляются.

### 9. Лидерборд

Назначение: показать результаты после вопроса или в конце квиза.

Ключевые элементы:

- Рейтинг участников.
- Nickname.
- Total score.
- Изменение позиции.
- Для организатора: Next question / Finish quiz.

Состояния:

- Loading: расчет очков.
- Empty: еще нет ответов.
- Error: ошибка загрузки результатов.
- Active: лидерборд отображает актуальные места.

### 10. История

Назначение: дать организатору посмотреть завершенные сессии и результаты.

Ключевые элементы:

- Список проведенных сессий.
- Дата, quiz title, количество участников.
- Финальный лидерборд.
- Детализация ответов по вопросам.

Состояния:

- Loading: загрузка истории.
- Empty: завершенных сессий пока нет.
- Error: ошибка загрузки истории.
- Active: история доступна для просмотра.

## User Flow 1: организатор создает и запускает квиз

1. Организатор открывает лендинг.
2. Переходит к регистрации или логину.
3. После входа попадает в кабинет организатора.
4. Нажимает создание квиза.
5. Заполняет title, description, category и default time limit.
6. Добавляет вопросы и варианты ответов.
7. Сохраняет квиз.
8. В кабинете нажимает `Start session`.
9. Видит лобби с 6-символьным кодом.
10. После подключения участников нажимает `Start quiz`.

## User Flow 2: участник подключается по коду и отвечает

1. Участник открывает лендинг.
2. Вводит 6-символьный код сессии.
3. Вводит nickname или авторизуется.
4. Попадает в лобби и видит статус ожидания.
5. После старта получает экран текущего вопроса.
6. Выбирает один или несколько вариантов.
7. Отправляет ответ до завершения таймера.
8. Видит промежуточное состояние ожидания результатов.

## User Flow 3: показ лидерборда и сохранение

1. После закрытия вопроса сервер считает correctness, response time и points.
2. На сцене открывается лидерборд текущей сессии.
3. Участники видят свой результат и место.
4. Организатор нажимает `Next question` или `Finish quiz`.
5. После завершения сессия получает статус `FINISHED`.
6. `endedAt`, ответы участников и итоговые баллы остаются в БД.
7. Организатор открывает историю и видит завершенную сессию.
